2. What Does "i.i.d." Mean?

You'll see the term i.i.d. a lot in statistics. It stands for independent and identically distributed — two properties that your data points need to have for most standard statistical formulas to work.

Independent

One observation doesn't influence or tell you anything about another.

Example: Flipping a coin — getting heads on flip 1 tells you nothing about flip 2. Each flip is independent.

Counter-example: Two requests from the same account — if one is slow (maybe the account's network is congested), the other is likely slow too. They are not independent.

Identically Distributed

Every observation is drawn from the same underlying process — same probability distribution, same rules.

Example: Every account in the experiment is drawn from the same pool of accounts, under the same conditions.

Counter-example: Mixing data from free-tier accounts and enterprise accounts without accounting for the difference — they come from different distributions.

Why does i.i.d. matter so much?

The formula Var(sample mean) = Var(data) / n assumes each data point contributes one "unit" of new information. But if data points are correlated (not independent), some of that information is redundant — like asking the same person the same question twice. The formula would say you have more precision than you actually do, leading to overconfident results.

That's why we aggregate to the randomization unit (e.g., account) — at that level, observations are i.i.d., and the formulas work correctly.

4. Clustering — Why It Matters

Independent (between clusters)

Account A and Account B are independent — assigned separately to control/treatment.

Each account = one randomization unit = one cluster.

Correlated (within a cluster)

Requests within Account A share network conditions, user behavior, etc.

Request a1 and a2 are not independent — they're correlated.

5. Linear vs. Nonlinear — Why Ratios Are Tricky

LINEAR Simple Mean

When randomization unit = analysis unit:

avg = (x₁ + x₂ + … + xₙ) / n = (1/n)·x₁ + (1/n)·x₂ + …

Multiply by constant, add up → linear!

Standard variance formula works.

NONLINEAR Ratio of Means

When we rewrite for a different randomization unit:

ratio = mean_numerator / mean_denominator

Division of two random variables → nonlinear!

Can't naively compute variance. Need the Delta Method.

What goes wrong without the Delta Method?
  • Variance too small → false confidence → Type I errors (false positives)
  • Variance too large → overcautious → Type II errors (miss real effects)

6. What Is the Delta Method?

Definition in plain English

The Delta Method is a technique for computing the approximate variance of a function of random variables — specifically when that function is nonlinear (like a ratio).

It works in three steps:

  1. Take your nonlinear function (e.g., X/Y) and approximate it as a linear function near the average values
  2. Because it's now linear, use the standard variance formula that only works for linear functions
  3. The result is an approximate (not exact) variance — good enough for practical use, especially with large sample sizes

That's it. The Delta Method is just a way to turn a hard problem (variance of a nonlinear function) into an easy problem (variance of a linear function) by using an approximation.

7. How the Delta Method Works — Step by Step

The setup: two worlds of variance

There are two kinds of functions you might want to compute variance for. One has a formula. The other doesn't.

Linear functions

A function like f(X, Y) = aX + bY — multiply by constants, add up.

There's an exact formula for variance:

Var(aX + bY) = a²·Var(X) + b²·Var(Y) + 2ab·Cov(X, Y)

This works because the function is a straight line — the spread of the output is a predictable, proportional transformation of the spread of the inputs.

Nonlinear functions (like a ratio)

A function like f(X, Y) = X / Y — division of two variables.

There is no exact formula for Var(X/Y). Small changes in Y (the denominator) can cause disproportionately large swings in the output. The linear variance rule simply doesn't apply.

The key insight: zoom in until the curve looks like a line

Think about any smooth curve — like a hill or a parabola. If you zoom in close enough to a single point on the curve, it starts to look like a straight line. This is actually the idea behind derivatives in calculus: the derivative at a point tells you the slope of that tiny straight line.

The Delta Method uses this insight. It says: "I can't compute variance for a curve, but I can for a straight line. So let me zoom in on the curve at the point where our data is centered (the mean), replace the curve with the straight line that matches it there, and compute variance on that line instead."

An analogy: the flat board on a hillside

Imagine you're standing on a curved hillside and want to know how tilted the ground is beneath your feet. The hill is curved, so there's no single "tilt" for the whole hill. But if you put a flat board under your feet, that board's angle approximates the tilt at your exact spot.

  • The curved hill = the nonlinear ratio function (X/Y)
  • The flat board = the linear approximation the Delta Method creates
  • Your feet = the mean values (where the data is centered)
  • The board's angle = the approximate variance we compute

The approximation is good when you're close to the mean (standing in one spot). It gets worse the farther out you go — which is why the Delta Method works well with large sample sizes, where sample means cluster tightly around the true mean.

The math behind it (jargon-free walkthrough)

Here's what the Delta Method actually does, step by step. Each step builds on the last:

Step 1 — "Taylor expand" = replace the curve with a line

We have the function f(X, Y) = X / Y. We pick a point to center on: the mean values (μ_X, μ_Y). Then we write an approximation that says:

f(X, Y) ≈ f(μ_X, μ_Y) + (slope in X direction) × (X − μ_X) + (slope in Y direction) × (Y − μ_Y)

This is called a first-order Taylor expansion. "First-order" means we only keep the straight-line part and throw away the curvature. The result is a linear function of X and Y — exactly the kind we know how to compute variance for.

Step 2 — Compute the slopes (partial derivatives)

The "slopes" in the formula above are just partial derivatives — they tell you how fast the output changes when you nudge one input while holding the other fixed:

  • Slope in X direction: If X (the numerator) goes up by 1, the ratio goes up by 1/μ_Y
  • Slope in Y direction: If Y (the denominator) goes up by 1, the ratio goes down by μ_X/μ_Y²

The denominator's slope is negative — which makes intuitive sense: if you divide by a bigger number, the result gets smaller.

Step 3 — Apply the linear variance formula

Now that we have a linear approximation with known slopes (a and b), we plug into the variance formula we already know:

Var(ratio) ≈ (1/μ_Y)² · Var(X) + (μ_X/μ_Y²)² · Var(Y) − 2 · (μ_X/μ_Y³) · Cov(X, Y)

This is the Delta Method formula. It's approximate (because we threw away the curvature in Step 1), but it's a very good approximation with large enough samples.
